01 — foundations

Matplotlib vs Seaborn — What's the Difference?

Python has two dominant visualization libraries. You'll use both — they complement each other perfectly.

Matplotlib

The foundation. Extremely flexible and powerful, but verbose. You control every pixel. Think of it as a blank canvas — you can paint anything, but you have to paint it yourself. Used when you need full customisation or precise layouts.

Seaborn

Built on top of Matplotlib. Beautiful defaults, statistical built-in features, and DataFrame-native. One line of Seaborn code equals 10-15 lines of raw Matplotlib. Use Seaborn for exploration and EDA — it's faster and produces publication-ready charts with minimal effort. Seaborn doesn't replace Matplotlib; it extends it.

python — setup
import matplotlib.pyplot as plt
import seaborn as sns
import pandas as pd
import numpy as np

# Load the Titanic dataset
url = "https://raw.githubusercontent.com/datasciencedojo/datasets/master/titanic.csv"
df = pd.read_csv(url)
df["Age"] = df["Age"].fillna(df["Age"].median())

# Set a clean Seaborn theme for all plots
sns.set_theme(style="whitegrid", palette="muted")
plt.rcParams["figure.dpi"] = 120    # sharper output in Colab
plt.rcParams["figure.figsize"] = (10, 5)
02 — bar chart

Bar Chart — Comparing Categories

Use a bar chart when you want to compare values across distinct categories. It's the most common chart type for a reason: it's immediately readable by anyone.

bar_chart.py 
# Matplotlib — full control
class_survival = (
  df.groupby("Pclass")["Survived"]
    .mean() * 100
)

fig, ax = plt.subplots()
ax.bar(
  class_survival.index,
  class_survival.values,
  color="#1a6eff",
  alpha=0.85,
  width=0.6
)
ax.set_xlabel("Passenger Class")
ax.set_ylabel("Survival Rate (%)")
ax.set_title("Survival by Class")
plt.show()

# Seaborn — one line!
sns.barplot(
  data=df, x="Pclass",
  y="Survived", hue="Sex"
)
plt.show()
The anatomy of a Matplotlib figure
Every Matplotlib chart has a Figure (the whole canvas) and one or more Axes (the actual plot area). Always use fig, ax = plt.subplots() then call methods on ax — this gives you full control and is the professional way to write Matplotlib code. The shortcut plt.bar() works but is harder to customise.

04 — scatter plot

Scatter Plot — Finding Relationships

Scatter plots show the relationship between two numeric variables. Each dot is one data point, plotted at coordinates (x, y). When dots cluster in a pattern — a line, a curve, a cluster — that's a correlation or a group worth investigating.

05 — histogram

Histogram — Understanding Distribution

A histogram shows how a single numeric variable is distributed — how many values fall in each range (bin). Unlike a bar chart which compares categories, a histogram reveals the shape of data: is it normal? Skewed? Are there two peaks (bimodal)? Are there outliers?

python
# Matplotlib histogram
fig, ax = plt.subplots()
ax.hist(df["Age"], bins=30,
       color="#8b5cf6", edgecolor="white")
ax.set_xlabel("Age")
ax.set_ylabel("Count")
ax.set_title("Age Distribution of Passengers")
plt.show()

# Seaborn histplot — adds KDE curve automatically
sns.histplot(
  data=df, x="Age",
  hue="Survived",     # split by survival!
  kde=True,            # smooth density curve
  bins=30, alpha=0.6
)
plt.title("Age Distribution by Survival")
plt.show()
Choosing the number of bins
Too few bins (e.g. 5) and you lose detail — the distribution looks flat and uninformative. Too many bins (e.g. 200) and you get noise — every tiny fluctuation shows as a spike. A good starting point is bins=30 for most datasets. Seaborn's default is usually sensible. Look at the shape: if it's too jagged, reduce bins; if it's too blocky, increase them.
06 — heatmap

Heatmap — Visualising Correlations

A correlation heatmap shows how every numeric column in your dataset relates to every other. Values range from -1 (perfect negative correlation) to +1 (perfect positive correlation). This is typically one of the last things you make before starting ML modelling.

python
# Correlation matrix heatmap — the standard EDA closer
numeric_df = df.select_dtypes(include="number")
corr = numeric_df.corr()

fig, ax = plt.subplots(figsize=(8, 6))
sns.heatmap(
  corr,
  annot=True,       # show values inside cells
  fmt=".2f",         # 2 decimal places
  cmap="coolwarm",  # blue=negative, red=positive
  center=0,          # 0 = neutral grey
  square=True,
  linewidths=0.5,
  ax=ax
)
ax.set_title("Titanic — Feature Correlation Matrix")
plt.show()

# Find the features most correlated with survival
print(corr["Survived"].sort_values(ascending=False))
Output — Correlation with Survived

Fare          0.257   ← higher fare = more survival
Parch         0.082
SibSp        -0.035
Age          -0.070

Reading the heatmap
A value close to +1 means the two variables move together — as one rises, so does the other. Close to -1 means they move opposite — as one rises, the other falls. Close to 0 means no linear relationship. In the Titanic data: Pclass has -0.338 correlation with Survived — a strong signal that class dramatically affected survival odds.
07 — when to use what

The Chart Selection Guide

The most common mistake beginners make is using the wrong chart for their data. Here's a quick reference:

Bar Chart
Comparing values across a few distinct categories (max ~15). Survival by class. Sales by region. Counts by label.
Line Chart
Showing change over an ordered sequence — time series, training epochs, model iterations. Never use for unordered categories.
Scatter Plot
Exploring the relationship between two numeric variables. Looking for correlations, clusters, or outliers in your data.
Histogram
Understanding how a single numeric column is distributed. Is it skewed? Are there outliers? Is it bimodal? Essential for EDA.
Heatmap
Visualising a matrix of values — most commonly a correlation matrix between all features before ML modelling.
Pie / Donut
Showing part-to-whole relationships — only when you have fewer than 5 categories and the total matters. Often a bar chart is clearer.
08 — mini project

Project: Titanic Visual Story

Let's create a 2×2 dashboard of four charts that tell the full Titanic story — all in one figure using plt.subplots().

python
import matplotlib.pyplot as plt
import seaborn as sns
import pandas as pd

url = "https://raw.githubusercontent.com/datasciencedojo/datasets/master/titanic.csv"
df = pd.read_csv(url)
df["Age"] = df["Age"].fillna(df["Age"].median())

sns.set_theme(style="whitegrid")

# 2x2 dashboard
fig, axes = plt.subplots(2, 2, figsize=(14, 10))
fig.suptitle("Titanic — The Data Story",
             fontsize=18, fontweight="bold")

# Chart 1 (top-left): Survival by Class
sns.barplot(
  data=df, x="Pclass", y="Survived",
  palette="Blues_r", ax=axes[0][0]
)
axes[0][0].set_title("Survival Rate by Class")
axes[0][0].set_ylabel("Survival Rate")
axes[0][0].set_xticklabels(["1st","2nd","3rd"])

# Chart 2 (top-right): Survival by Gender
sns.barplot(
  data=df, x="Sex", y="Survived",
  palette=["#ff6b9d","#1a6eff"],
  ax=axes[0][1]
)
axes[0][1].set_title("Survival Rate by Gender")
axes[0][1].set_ylabel("Survival Rate")

# Chart 3 (bottom-left): Age distribution
sns.histplot(
  data=df, x="Age", hue="Survived",
  kde=True, bins=30, alpha=0.6,
  ax=axes[1][0]
)
axes[1][0].set_title("Age Distribution by Survival")

# Chart 4 (bottom-right): Fare vs Age scatter
sns.scatterplot(
  data=df, x="Age", y="Fare",
  hue="Survived", alpha=0.5,
  ax=axes[1][1]
)
axes[1][1].set_title("Age vs Fare (by Survival)")

plt.tight_layout()
plt.show()
plt.tight_layout() — always use it
When you have multiple subplots, labels and titles can overlap. plt.tight_layout() automatically adjusts the spacing between subplots so nothing overlaps. Call it every time before plt.show() when using subplots. It takes one second and makes your charts look professional.
