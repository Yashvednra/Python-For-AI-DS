01 — list

List — Your Swiss Army Knife

A list is an ordered, mutable sequence. You can add, remove, sort, and slice it freely. It's the most commonly used data structure in Python, and in data science you'll use lists constantly — to store feature names, rows of data, model predictions, and more.

List [   ]
  • ordered
  • mutable
  • allows duplicates

Ordered means items have a position (index), starting from 0. names[0] gives you the first item.

Mutable means you can change, add, or remove items after the list is created.

Allows duplicates — the same value can appear multiple times.

  • Access items by index: lst[0], lst[-1]
  • Slice: lst[1:4], lst[:3]
  • Add: .append(), .insert()
  • Remove: .remove(), .pop()
  • Sort: .sort(), sorted(lst)
  • Length: len(lst)
list_basics.py 
# Create a list
names = ["Arjun", "Priya",
         "Rohan", "Sneha"]

# Access by index
print(names[0])     # Arjun
print(names[-1])    # Sneha

# Slice [start:stop]
print(names[1:3])   # ['Priya','Rohan']

# Add / remove
names.append("Dev")
names.remove("Rohan")

# Sort
names.sort()
print(names)

Practical — Student names: sort and slice

python
# Classroom student list
students = [
    "Priya", "Arjun", "Sneha",
    "Rohan", "Dev",   "Ananya"
]

print(f"Total students: {len(students)}")

# Sort alphabetically
students.sort()
print("Sorted:", students)

# Slice — first 3 students
print("Top 3:", students[:3])

# Slice — last 2 students
print("Last 2:", students[-2:])

# Add a new student
students.append("Vikram")
print("After adding:", students)

# Check membership
print("Arjun in list:", "Arjun" in students)
Output
Total students: 6
Sorted: ['Ananya', 'Arjun', 'Dev', 'Priya', 'Rohan', 'Sneha']
Top 3: ['Ananya', 'Arjun', 'Dev']
Last 2: ['Rohan', 'Sneha']
After adding: ['Ananya', 'Arjun', 'Dev', 'Priya', 'Rohan', 'Sneha', 'Vikram']
Arjun in list: True
Indexing tip
Python lists use zero-based indexing — the first item is at index 0, not 1. Negative indices count from the end: list[-1] is always the last item. In data science you'll use df.iloc[0] and similar — same concept.
02 — tuple

Tuple — The Immutable Guardian

A tuple is exactly like a list — ordered, allows duplicates — except it is immutable: once created, it cannot be changed. You can't add, remove, or modify items.

Tuple (   )
  • ordered
  • immutable
  • allows duplicates

Immutable means the data is locked after creation. Trying to change a tuple raises a TypeError.

When to use tuples: whenever your data should not change — GPS coordinates, RGB colours, dates, configuration values, function return values with multiple outputs.

  • Access by index just like a list
  • Faster than lists (less overhead)
  • Can be used as dict keys (lists cannot)
  • Unpack easily: x, y = (1, 2)
  • Count occurrences: .count()
  • Find index: .index()
tuple_basics.py 
# GPS coordinates — never change
delhi  = (28.6139, 77.2090)
mumbai = (19.0760, 72.8777)

# Unpack into variables
lat, lon = delhi
print(f"Lat: {lat}")

# RGB colour — fixed
saffron = (255, 153, 51)

# This would cause a TypeError:
# delhi[0] = 30  ← cannot modify!

# But you CAN read it
print(delhi[0])   # 28.6139
List vs Tuple — the mental model
Use a list for a shopping cart — items come and go. Use a tuple for a date of birth — it never changes. In data science, tuples are commonly used for shape (e.g., a dataset has shape (1000, 12) — 1000 rows, 12 columns). You'll see this in NumPy and Pandas constantly.
03 — dictionary

Dictionary — The Fast Lookup Table

A dictionary stores data as key-value pairs. Instead of accessing items by position (index), you access them by a meaningful key — like looking up a word in a real dictionary.

Dict { key : value }
  • key→value
  • fast lookup
  • unique keys

Keys must be unique and immutable (strings, numbers, tuples). Values can be anything — numbers, strings, lists, even other dicts.

Lookup is extremely fast — Python uses a hash table internally. Looking up a key in a dict with 1 million entries is just as fast as looking it up in a 10-entry dict.

  • Access: d["key"] or d.get("key")
  • Add/update: d["key"] = value
  • Delete: del d["key"]
  • All keys: d.keys()
  • All values: d.values()
  • Both: d.items()
dict_basics.py 
# Student scores
scores = {
    "Arjun":  92,
    "Priya":  87,
    "Sneha":  95
}

# Access by key
print(scores["Priya"])  # 87

# Safe access with .get()
print(scores.get("Dev", 0))  # 0

# Add new student
scores["Rohan"] = 78

Practical — Student scores + looping with .items()

python
# Dictionary of student scores
scores = {
    "Arjun":  92,
    "Priya":  87,
    "Sneha":  95,
    "Rohan":  78,
    "Dev":    83
}

# Loop through key-value pairs
print("Score Report:")
print("-" * 25)
for name, score in scores.items():
    grade = "A" if score >= 90 else "B" if score >= 80 else "C"
    print(f"  {name:<8}: {score} → {grade}")

# Summary stats
all_scores = list(scores.values())
print(f"\nHighest : {max(all_scores)}")
print(f"Average : {sum(all_scores)/len(all_scores):.1f}")
Output
Score Report:
-------------------------
  Arjun   : 92 → A
  Priya   : 87 → B
  Sneha   : 95 → A
  Rohan   : 78 → C
  Dev     : 83 → B

Highest : 95
Average : 87.0
Data Science connection
In data science, dictionaries are everywhere. A Pandas DataFrame column is essentially a dict under the hood. When you do df["name"] to select a column — that's dict-style access. Model hyperparameters are stored as dicts: {"n_estimators": 100, "max_depth": 5}. Master dicts and you'll understand a lot of ML library syntax naturally.
04 — set

Set — The Duplicate Destroyer

A set is an unordered collection with no duplicates. If you try to add the same value twice, a set simply ignores the second one. This makes sets perfect for finding unique values, deduplication, and fast membership testing.

Set {   }
  • unordered
  • no duplicates
  • mutable

Unordered means there's no index. You can't do my_set[0]. Sets are not for accessing individual items — they're for operations on the whole collection.

No duplicates is the key feature. Pass any list with repeats into set() and you instantly get back only unique values.

  • Add: .add()
  • Remove: .remove(), .discard()
  • Union: a | b or a.union(b)
  • Intersection: a & b
  • Difference: a - b
  • Membership: "x" in my_set (very fast)
set_basics.py 
# Duplicates are ignored
cities = {"Delhi", "Mumbai",
          "Delhi", "Chennai"}
print(cities)
# {'Mumbai','Delhi','Chennai'}

# Convert list → set to deduplicate
tags = ["python","data","python"]
unique = set(tags)
# {'python', 'data'}

# Fast membership test
print("Delhi" in cities)  # True

Practical — Find unique visitors from a log

python
# Website visitor log — many repeat visits
visitor_log = [
    "arjun", "priya", "arjun",
    "dev",   "sneha", "priya",
    "arjun", "rohan", "sneha"
]

print(f"Total visits : {len(visitor_log)}")

# Unique visitors — one line!
unique_visitors = set(visitor_log)
print(f"Unique visitors: {len(unique_visitors)}")
print(unique_visitors)

# Set operations — practical examples
monday    = {"arjun", "priya", "sneha"}
tuesday   = {"priya", "dev",   "rohan"}

# Who visited BOTH days?
print("Both days:", monday & tuesday)

# Who visited on Monday but NOT Tuesday?
print("Mon only :", monday - tuesday)

# All unique visitors across both days?
print("All unique:", monday | tuesday)
Output
Total visits : 9
Unique visitors: 5
{'arjun', 'dev', 'priya', 'rohan', 'sneha'}
Both days: {'priya'}
Mon only : {'arjun', 'sneha'}
All unique: {'arjun', 'dev', 'priya', 'rohan', 'sneha'}
When sets shine in data science
Sets are used constantly in data cleaning. Removing duplicate rows, finding which categories appear in column A but not column B, or checking if two feature sets overlap — all set operations. set(df["city"].unique()) is a pattern you'll write dozens of times.
05 — decision guide

When to Use Which?

The hard part isn't the syntax — it's choosing the right structure. Here's a practical guide:

List
  • Storing a sequence of items in order
  • Items you'll add, remove, or change
  • Collecting results in a loop
  • Feature names in a ML model
  • Rows of CSV data before processing
Tuple
  • Data that should never change
  • GPS coordinates, RGB colours, dates
  • Returning multiple values from a function
  • Dataset shape: (1000, 12)
  • As dictionary keys (lists can't be keys)
Dict
  • Mapping names → scores, IDs → records
  • Counting frequencies of items
  • Model hyperparameter configs
  • JSON-like structured data
  • Any time you look up by a label
Set
  • Finding unique values in a column
  • Removing duplicates from a list
  • Checking if value exists (very fast)
  • Finding common elements between two lists
  • Data validation — allowed values
