01 — what is EDA

What is Exploratory Data Analysis?

Exploratory Data Analysis (EDA) is the process of investigating a dataset to understand its structure, find patterns, detect anomalies, and test hypotheses — before any formal modelling. The term was coined by statistician John Tukey in 1977 and remains the foundation of every serious data science project.

Think of it as detective work. You've been handed a case file (your dataset). Before making any conclusions, you look at the evidence from every angle. You ask questions. You build a picture. You form theories — then check them against the data.

1
Understand the dataset structure
What does each column mean? How many rows? What data types? Are there nulls?
shape · info() · dtypes · isnull().sum()
2
Univariate analysis — one column at a time
How is each variable distributed? What's the range? Any outliers? Any skew?
describe() · histplot() · value_counts() · boxplot()
3
Bivariate analysis — two columns together
How do pairs of variables relate? Is there correlation? Do categories differ?
scatterplot() · boxplot(x,y) · groupby().mean() · corr()
4
Multivariate analysis — many columns together
How do multiple variables interact? What patterns emerge when you combine features?
heatmap(corr()) · pairplot() · hue= parameter
5
Document findings and hypotheses
Write down what you found. What features matter? What needs cleaning? What will your model use?
Markdown cells in Colab · written summary
02 — setup

Setting Up Your EDA Notebook

python — eda_setup.py
# ── EDA Standard Imports ──────────────────────────────
import pandas as pd
import numpy as np
import matplotlib.pyplot as plt
import seaborn as sns
import warnings
warnings.filterwarnings("ignore")

# Style setup
sns.set_theme(style="whitegrid", palette="muted", font_scale=1.1)
plt.rcParams["figure.dpi"]    = 120
plt.rcParams["figure.figsize"] = (10, 5)

# Load and clean the Titanic dataset
url = "https://raw.githubusercontent.com/datasciencedojo/datasets/master/titanic.csv"
df = pd.read_csv(url)
df["Age"]  = df["Age"].fillna(df["Age"].median())
df  = df.dropna(subset=["Embarked"])
df  = df.drop(columns=["Cabin"])

print(f"Dataset ready: {df.shape} — {df.isnull().sum().sum()} nulls remaining")

03 — univariate analysis

Univariate Analysis — One Variable at a Time

Univariate means you examine each column in isolation. The goal is to understand the distribution, central tendency, spread, and outliers of each feature. This is where you first spot problems and surprises.

Numeric columns — describe + histogram

python
# ── Step 1: Get stats for all numeric columns ─────────
print(df.describe())

# ── Step 2: Visualise each numeric distribution ───────
numeric_cols = ["Age", "Fare", "SibSp", "Parch"]

fig, axes = plt.subplots(2, 2, figsize=(14, 8))
axes = axes.flatten()

for i, col in enumerate(numeric_cols):
    sns.histplot(
        data=df, x=col, kde=True,
        ax=axes[i], color="#d4881a"
    )
    axes[i].set_title(f"{col} Distribution")
    axes[i].axvline(df[col].mean(),
        color="red", linestyle="--",
        label=f"Mean: {df[col].mean():.1f}")
    axes[i].legend(fontsize=9)

plt.suptitle("Titanic — Numeric Distributions",
             fontsize=14, fontweight="bold")
plt.tight_layout()
plt.show()
Age is roughly normally distributed (mean ~29.7). Fare is heavily right-skewed — most passengers paid cheap tickets but a few paid enormous sums. SibSp and Parch are both heavily concentrated at 0 — most passengers were travelling alone.
→ Fare needs log-transformation before modelling. SibSp & Parch may be combined into a single "FamilySize" feature.

Categorical columns — value counts + bar charts

python
# All categorical columns at once
cat_cols = ["Sex", "Pclass", "Embarked"]

fig, axes = plt.subplots(1, 3, figsize=(14, 4))

for i, col in enumerate(cat_cols):
    counts = df[col].value_counts()
    axes[i].bar(counts.index.astype(str),
               counts.values, color="#d4881a",
               alpha=0.8)
    axes[i].set_title(f"{col} Counts")

    # Add count labels on top of bars
    for j, v in enumerate(counts.values):
        axes[i].text(j, v+5, str(v),
                    ha="center", fontsize=10)

plt.tight_layout()
plt.show()

# Detailed look at a single column
print(df["Pclass"].value_counts(normalize=True).round(3))

Univariate insight for modelling
Heavily imbalanced or skewed columns are a warning sign. The Fare column's extreme skew (max £512 when median is £14) will distort any model that assumes normal distributions. Flag these for transformation. The Pclass imbalance (55% 3rd class) means your model will see far more 3rd class examples — worth noting before evaluation.

05 — multivariate analysis

Multivariate Analysis — The Full Picture

Now we look at three or more variables together. This reveals interaction effects — situations where the effect of one variable depends on another. These are often the richest insights in EDA.

Interaction: Sex × Class × Survival

            cmap="YlOrRd", ax=axes[1],

06 — engineering from EDA

Feature Engineering Insights from EDA

Good EDA doesn't just describe data — it suggests new features to create. Here are the features our Titanic EDA suggests we should engineer before modelling:

python
# ── Feature 1: FamilySize ─────────────────────────────
# SibSp + Parch + 1 (the passenger themselves)
df["FamilySize"] = df["SibSp"] + df["Parch"] + 1
df["IsAlone"]   = (df["FamilySize"] == 1).astype(int)

# Check: solo travellers vs group survivors
print(df.groupby("IsAlone")["Survived"].mean())

# ── Feature 2: IsChild ────────────────────────────────
df["IsChild"] = (df["Age"] < 10).astype(int)
print(df.groupby("IsChild")["Survived"].mean())

# ── Feature 3: Title from Name ────────────────────────
df["Title"] = (df["Name"]
    .str.extract(r" ([A-Za-z]+)\.")[0])

# Group rare titles
rare = ["Lady","Countess","Capt","Col",
         "Don","Rev","Dr","Major","Sir","Jonkheer"]
df["Title"] = df["Title"].replace(rare, "Rare")
df["Title"] = df["Title"].replace(
    {"Mlle":"Miss", "Ms":"Miss", "Mme":"Mrs"})

print(df.groupby("Title")["Survived"]
        .agg(["mean","count"])
        .round(3))

This is a feature we'd never know to create without EDA. The best features often come from domain understanding combined with data observation — not from automated feature selection.
07 — the EDA report

The Five Key Findings

Every EDA ends with a written summary. This is where you transform raw numbers into insight — the kind of summary that tells a non-technical person what the data is saying.

What is the overall survival rate?
Only 38% of passengers survived
889 passengers total after cleaning. 342 survived. The stark majority — 547 people — did not make it off the ship.
df["Survived"].mean() = 0.384
Which single feature predicts survival best?
Sex is the strongest predictor by far
Women survived at 74.2% — nearly 4× the male rate of 18.9%. The protocol "women and children first" is clearly visible in the data.
Gap: +55.3 percentage points
Did wealth affect survival odds?
Dramatically — class was life or death
1st class: 63% survival. 2nd class: 47%. 3rd class: 24%. Passengers who paid higher fares (corr +0.257 with survival) had access to better cabin locations and lifeboats.
1st class 2.6× more likely to survive than 3rd
Did travelling alone hurt survival chances?
Yes — solo travellers survived less
Passengers travelling alone had a 30.4% survival rate vs 50.6% for those with family. Being in a group meant people helped each other reach lifeboats.
IsAlone = 1 → survival drops 20 points
The three strongest predictors of Titanic survival are: (1) Sex — female passengers had a 74% rate vs 19% for males. (2) Pclass — 1st class had 63% survival vs 24% for 3rd class. (3) The interaction of Sex × Pclass — 1st class women had a 96.8% survival rate, the highest of any subgroup. These three variables, plus engineered features (FamilySize, IsChild, Title), form the core of a predictive model.
→ Recommended features for modelling: Sex, Pclass, Age, Fare, FamilySize, IsAlone, IsChild, Title
