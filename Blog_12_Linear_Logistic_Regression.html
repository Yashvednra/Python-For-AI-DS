01 — the problem

Two Types of Prediction

Machine learning is about teaching computers to predict. But there are two fundamentally different kinds of predictions:

Linear Regression: Predicting a number. "What will this house cost?" "How many calls will we get tomorrow?" "What's the temperature in 3 hours?" The output is continuous — it can be any number.

Logistic Regression: Predicting a category (usually yes/no). "Will this patient survive surgery?" "Is this email spam?" "Did this customer churn?" The output is discrete — one of a fixed set of options.

Both are supervised learning — you teach the model with examples that have both the question and the correct answer. After learning, the model predicts on new questions it's never seen.

02 — linear regression theory

Linear Regression: y = mx + b

You've seen this equation since algebra class. In machine learning, it means exactly what you think:

y = mx + b

y is what we're trying to predict (house price). x is the feature we're using (square footage). m is the slope — how much does y change when x goes up by 1? b is the intercept — what's the predicted value when x is zero?

During training, the model learns the best values of m and b to fit your data. During prediction, it plugs in a new value of x and calculates y.

Real datasets have many features, not one
With multiple features (square footage, bedrooms, age, location), the equation becomes y = m₁x₁ + m₂x₂ + ... + mₙxₙ + b. Each feature gets its own slope. The model learns all of them.
Loss Function for Regression

Mean Squared Error (MSE)

How do we know if our line is good? We measure the distance from each data point to the line, square it (to punish big errors more), and average those squared errors.

MSE = (1/n) × Σ(predicted - actual)²

The goal of training is to find m and b that minimize MSE. A line that passes through all points has MSE = 0. A bad line might have MSE = 10000. During training, the model adjusts the slope and intercept down this loss curve until it finds the minimum.

03 — logistic regression theory

Logistic Regression: Predicting Categories

Despite the name, logistic regression is not linear — it's a classification algorithm. It solves the problem: "Given features, what's the probability this belongs to category X?"

The key innovation is the sigmoid function, which squashes any number into a probability between 0 and 1:

P(y=1) = 1 / (1 + e^(-z))

Where z is the linear combination m₁x₁ + m₂x₂ + ... + b. If the probability is above 0.5, we predict "yes" (1). If below 0.5, we predict "no" (0).

Why sigmoid, not linear?
A linear model can predict probabilities outside [0, 1], which is nonsense. The sigmoid function is mathematically elegant: it's always between 0 and 1, it's differentiable (trainable), and it naturally represents increasing confidence as z gets more extreme.
Loss Function for Classification

Log Loss (Binary Crossentropy)

For classification, we use log loss instead of MSE. It harshly penalizes confident wrong predictions.

Log Loss = -(1/n) × Σ[y·log(p) + (1-y)·log(1-p)]

For each prediction: if the true label is 1, we measure -log(p) (large loss if p is near 0, small loss if p is near 1). If the true label is 0, we measure -log(1-p) (punishes predicting 1 confidently when the answer is 0).

The model learns weights that minimize log loss. A perfect classifier has loss ≈ 0. A random guesser (p = 0.5 always) has loss ≈ 0.69.

04 — linear regression in code

Linear Regression: House Prices

Let's predict house prices from square footage using a real dataset. We'll train the model, visualize the regression line, and evaluate its performance.

python
import pandas as pd
import numpy as np
from sklearn.linear_model import LinearRegression
from sklearn.metrics import mean_squared_error, r2_score
import matplotlib.pyplot as plt

# Load a simple house prices dataset
data = {
    "sqft": [1000, 1500, 2000, 2500, 3000, 3500],
    "price": [200000, 300000, 350000, 400000, 500000, 550000]
}

df = pd.DataFrame(data)

# Prepare features (X) and target (y)
X = df[["sqft"]]  # 2D array required by sklearn
y = df["price"]

# Train the model
model = LinearRegression()
model.fit(X, y)

# Get model parameters
print(f"Slope (m)     : {model.coef_[0].:.2f}")
print(f"Intercept (b) : {model.intercept_.:.2f}")

# Make predictions
y_pred = model.predict(X)

# Evaluate: MSE and R²
mse  = mean_squared_error(y, y_pred)
rmse = np.sqrt(mse)
r2   = r2_score(y, y_pred)

print(f"MSE  : {mse:.0f}")
print(f"RMSE : ${rmse:.0f}")
print(f"R²   : {r2:.3f}")

# Predict on new data
new_sqft = [[2200]]
predicted_price = model.predict(new_sqft)
print(f"\nPredicted price for 2200 sqft: ${predicted_price[0]:.0f}")
Output
Slope (m) : 142.86
Intercept (b) : 71428.57
MSE : 1020408163
RMSE : $31944
R² : 0.991

Predicted price for 2200 sqft: $385714

The model learned a slope of ~$143 per square foot. An R² of 0.991 means the model explains 99.1% of the variance in house prices — excellent fit!

python
# ── THE KEY: Visualize the regression line ──────────────
plt.figure(figsize=(10, 6))

# Plot actual data points
plt.scatter(X, y, color="orange", s=100, alpha=0.7, label="Actual prices")

# Plot the fitted regression line
plt.plot(X, y_pred, color="red", linewidth=2, label="Fitted line (y = 142.86x + 71428.57)")

plt.xlabel("Square Footage")
plt.ylabel("Price ($)")
plt.title("Linear Regression: House Prices vs Square Footage")
plt.legend()
plt.grid(True, alpha=0.3)
plt.tight_layout()
plt.show()
Always visualize the regression line
This is the most intuitive proof that your model "learned" something. If the line passes through the data points and follows their trend, your model captured the relationship. If the line looks random or skewed, something is wrong. Visualization is often more telling than any metric.
05 — logistic regression in code

Logistic Regression: Titanic Survival

Now we'll predict whether a passenger survived the Titanic. Our features: age, sex, and passenger class. Our target: survived (1) or not (0).

python
import pandas as pd
from sklearn.linear_model import LogisticRegression
from sklearn.metrics import accuracy_score, confusion_matrix, classification_report
from sklearn.preprocessing import LabelEncoder

# Load Titanic dataset
url = "https://raw.githubusercontent.com/datasciencedojo/datasets/master/titanic.csv"
df = pd.read_csv(url)

# Quick cleaning: drop rows with missing Age
df = df.dropna(subset=["Age"])

# Encode Sex column: male=1, female=0
le = LabelEncoder()
df["Sex_encoded"] = le.fit_transform(df["Sex"])  # female=0, male=1

# Features and target
features = ["Age", "Pclass", "Sex_encoded"]
X = df[features]
y = df["Survived"]

# Train logistic regression
model = LogisticRegression(max_iter=1000)
model.fit(X, y)

# Make predictions
y_pred = model.predict(X)
y_pred_proba = model.predict_proba(X)  # Probability of each class

# Accuracy score
accuracy = accuracy_score(y, y_pred)
print(f"Accuracy: {accuracy:.3f} ({accuracy*100:.1f}%)")
Output
Accuracy: 0.776 (77.6%)
python
# ── Confusion Matrix ──────────────────────────────────
cm = confusion_matrix(y, y_pred)

print("Confusion Matrix:")
print(cm)
print()

# Parse the matrix
tn, fp, fn, tp = cm.ravel()

print("Interpretation:")
print(f"True Negatives  (TN) : {tn} (didn't survive, correctly predicted didn't survive)")
print(f"False Positives (FP) : {fp} (didn't survive, but we said survived) — WRONG")
print(f"False Negatives (FN) : {fn} (survived, but we said didn't) — WRONG")
print(f"True Positives  (TP) : {tp} (survived, correctly predicted survived)")
print()

# Calculate metrics
precision = tp / (tp + fp)  # Of those we said survived, how many actually did?
recall    = tp / (tp + fn)     # Of those who actually survived, how many did we find?
f1        = 2 * (precision * recall) / (precision + recall)

print(f"Precision: {precision:.3f}")
print(f"Recall   : {recall:.3f}")
print(f"F1 Score : {f1:.3f}")
Output

F1 Score : 0.771

Our model correctly predicted survival in 77.6% of cases. The confusion matrix shows that it's nearly balanced between false positives (45) and false negatives (47) — neither error dominates.

What the metrics mean: Precision answers "Of those we predicted would survive, how many actually did?" (77.5%). Recall answers "Of those who actually survived, how many did we catch?" (76.7%). The F1 score (0.771) is the harmonic mean — it balances both.

Accuracy is not always the right metric
If 95% of emails are not spam, a model that predicts "never spam" would be 95% accurate but useless. Use precision/recall when false positives and false negatives have different costs. Use F1 when you want to balance both. Always visualize with a confusion matrix first.
06 — model comparison

When to Use Each

Linear Regression when:

  • Your target is a continuous number (price, temperature, revenue)
  • You want to understand the magnitude of change (each unit of X causes Y units of change in the target)
  • The relationship is roughly linear (a straight line through the scatter plot makes sense)

Logistic Regression when:

  • Your target is a category (yes/no, survived/died, spam/not spam)
  • You want probabilities (confidence levels) alongside predictions
  • Your classes are roughly separable by a linear boundary

Both are linear models — they assume a linear decision boundary. For more complex, nonlinear relationships, you'll need decision trees, neural networks, or other advanced methods. But linear and logistic regression are the foundation: simple, interpretable, and often sufficient.

07 — next steps

Beyond the Basics

Once you've mastered these two algorithms, you're ready for:

  • Regularization (Ridge, Lasso): Prevent overfitting by penalizing large coefficients
  • Cross-validation: Reliably estimate performance on unseen data
  • Feature scaling: Some algorithms perform better when features are normalized
  • Feature engineering: Create new features (polynomial terms, interactions) to capture complex relationships
  • Multi-class classification: Extend logistic regression to predict 3+ categories
  • Decision trees & ensemble methods: Nonlinear models for trickier problems

Key Takeaways
Linear Regression learns y = mx + b to predict continuous numbers, minimizing MSE. Logistic Regression uses the sigmoid function to predict probabilities, minimizing log loss. Both are interpretable, trainable, and foundational. Always visualize: plot the regression line for linear, plot the confusion matrix and probability distribution for classification. These visuals are often more informative than any single number.
