00 — why functions
The core idea

Stop Repeating Yourself

Here's a pattern every beginner falls into: you write the same calculation in three places. Then the formula changes. Now you have to fix it in three places — and you forget one. This is how bugs are born.

Functions are Python's answer to this. You write the logic once, give it a name, and call it anywhere. And modules? They're collections of functions someone else already wrote — so you don't have to.

💡 The Golden Rule of Functions

If you write the same code more than once, it should be a function. If you write it more than never, it should probably be a function. Functions make code readable, testable, and reusable.

Before vs After — See the Difference Instantly

This is the exact "aha moment" the blog tip mentioned. Look at the same problem — with and without a function:

❌  Without a function
# Calculate BMI for 3 people
# (imagine doing this for 1000!)

h1 = 1.75
w1 = 70
bmi1 = w1 / (h1 ** 2)
print(f"BMI: {bmi1:.2f}")

h2 = 1.60
w2 = 55
bmi2 = w2 / (h2 ** 2)
print(f"BMI: {bmi2:.2f}")

h3 = 1.82
w3 = 90
bmi3 = w3 / (h3 ** 2)
print(f"BMI: {bmi3:.2f}")

# 😩 Repeated 3 times already.
# Change formula? Update 3 places.
✅  With a function
# Define once.
def calculate_bmi(weight, height):
    return weight / (height ** 2)

# Call it everywhere. Clean.
print(calculate_bmi(70, 1.75))
print(calculate_bmi(55, 1.60))
print(calculate_bmi(90, 1.82))

# ✅ Formula changes? Fix ONE line.
# ✅ Works for 1000 people too.
# ✅ Name tells you what it does.
# ✅ Easy to test in isolation.

The def Keyword

Every function in Python starts with def — short for define. Here's the complete anatomy of a function:

// anatomy of a Python function
def greet(name, greeting="Hello"):
    """Says hello to someone. (This is a docstring!)"""
    message = f"{greeting}, {name}! 👋"
    return message

Practical: Full BMI Calculator

Let's build a proper BMI calculator with a category label and default unit system:

bmi_calculator.py 
def calculate_bmi(weight, height):
    """
    Calculate Body Mass Index (BMI).
    weight: in kilograms
    height: in meters
    """
    bmi = weight / (height ** 2)
    return round(bmi, 2)


def bmi_category(bmi):
    """Return the WHO BMI category label."""
    if bmi < 18.5:
        return "Underweight 🔵"
    elif bmi < 25.0:
        return "Normal weight ✅"
    elif bmi < 30.0:
        return "Overweight 🟡"
    else:
        return "Obese 🔴"


def bmi_report(name, weight, height):
    """Print a full BMI report for a person."""
    bmi = calculate_bmi(weight, height)
    cat = bmi_category(bmi)
    print(f"─────────────────────────")
    print(f"  Name   : {name}")
    print(f"  Weight : {weight} kg")
    print(f"  Height : {height} m")
    print(f"  BMI    : {bmi}")
    print(f"  Status : {cat}")
    print(f"─────────────────────────")


# Call the function for multiple people
bmi_report("Arjun", 70, 1.75)
bmi_report("Priya", 55, 1.60)
bmi_report("Rahul", 95, 1.72)

02 — arguments
default · keyword args

Default Arguments

Default arguments let callers skip parameters they don't need to customize — the function uses a sensible fallback instead.

default_args.py 
def greet_user(name, greeting="Hello", punctuation="!"):
    """Greet someone with a customizable message."""
    print(f"{greeting}, {name}{punctuation}")


# All 3 ways to call the same function:
greet_user("Sneha")                          # uses both defaults
greet_user("Dev", "Good morning")            # custom greeting
greet_user("Ananya", "Namaste", "🙏")       # all custom

# Keyword arguments — order doesn't matter!
greet_user(punctuation=".", name="Ravi", greeting="Hey")
▶ Output
Hello, Sneha!
Good morning, Dev!
Namaste, Ananya 🙏
Hey, Ravi.
🔵 Keyword arguments = self-documenting code

When you write greet_user(name="Ravi", greeting="Hey"), the function call reads like English. This is especially valuable in data science where functions often have 10+ parameters.

⚠️ Rule: defaults come last

Parameters with default values must always come after parameters without defaults. def fn(name="A", age) is illegal — Python won't know what to assign. Always: def fn(required, optional="default")

03 — modules
import · standard library

What Is a Module?

A module is just a Python file containing functions, variables, and classes. Instead of writing every tool from scratch, you import them. Python ships with hundreds of modules already installed — the standard library.

🔧
Built-in Functions
Always available. No import needed. These are Python's core tools.
print() · len() · range() · type()
📦
Standard Library
Ships with Python. Import and use instantly. Hundreds of modules.
math · random · os · datetime · json
🌐
Third-Party
Written by the community. Install via pip, then import.
numpy · pandas · sklearn · tensorflow

Practical: The math module

using_math.py 
import math

# Square roots — you'll use this constantly in ML!
print(math.sqrt(144))       # → 12.0
print(math.sqrt(2))         # → 1.4142135623...

# Useful constants
print(math.pi)              # → 3.141592653589793
print(math.e)               # → 2.718281828459045

# Rounding & ceiling
print(math.floor(4.9))     # → 4   (rounds down)
print(math.ceil(4.1))      # → 5   (rounds up)
print(math.pow(2, 10))     # → 1024.0
print(math.log(math.e))    # → 1.0  (natural log)

# Real use: Euclidean distance formula
def distance(x1, y1, x2, y2):
    return math.sqrt((x2-x1)**2 + (y2-y1)**2)

print(distance(0, 0, 3, 4))  # → 5.0 (classic 3-4-5 triangle)

Practical: The random module — Dice Roller

dice_roller.py 
import random

def roll_dice(sides=6, num_dice=1):
    """
    Roll any number of dice with any number of sides.
    Default: 1 standard six-sided die.
    """
    results = []
    for _ in range(num_dice):
        roll = random.randint(1, sides)
        results.append(roll)
    return results


def dice_game():
    """A simple two-player dice game."""
    print("🎲 Welcome to the Dice Game!")
    print("─" * 30)

    players = ["Player 1", "Player 2"]
    scores  = {}

    for player in players:
        roll   = roll_dice(sides=6, num_dice=2)  # 2 dice each
        total  = sum(roll)
        scores[player] = total
        print(f"{player}: rolled {roll} → total = {total}")

    print("─" * 30)
    winner = max(scores, key=scores.get)

    if scores["Player 1"] == scores["Player 2"]:
        print("🤝 It's a tie!")
    else:
        print(f"🏆 {winner} wins!")


# --- Run the game ---
dice_game()

# --- Explore more random functions ---
print(random.choice(["rock", "paper", "scissors"]))
print(random.random())          # float between 0.0 and 1.0
print(random.randint(1, 100))  # random integer in range

04 — import styles
import · from · as

Four Ways to Import

import_styles.py 
# Style 1 — import the whole module
import math
math.sqrt(16)          # must prefix with module name

# Style 2 — import specific function (no prefix needed)
from math import sqrt, pi
sqrt(16)               # ✓ cleaner!
print(pi)

# Style 3 — alias (very common in data science!)
import numpy as np     # you'll write this in every DS notebook
import pandas as pd    # industry standard aliases
import random as rnd

# Style 4 — import everything (⚠ use sparingly)
from math import *     # pollutes namespace — avoid in production
⚠️ Never use import * in real projects

When you do from math import *, you import everything — including names that might clash with your own variables. Stick to explicit imports. Your future self (and teammates) will thank you.
