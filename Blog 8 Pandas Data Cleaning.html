A machine learning model trained on dirty data — missing values, wrong data types, duplicates, inconsistent text — will produce garbage predictions. Cleaning data is not glamorous. But it is the difference between models that work and models that fail.

02 — missing values

Handling Missing Values

Missing values appear as NaN (Not a Number) in Pandas. They happen because data wasn't collected, was lost, or was never applicable. Your first job is to find them, understand why they're missing, and decide what to do.

Step 1 — Find Missing Values

python
import pandas as pd
import numpy as np

url = "https://raw.githubusercontent.com/datasciencedojo/datasets/master/titanic.csv"
df = pd.read_csv(url)

# Count nulls per column
print(df.isnull().sum())

# Percentage missing — more actionable
missing_pct = (df.isnull().sum() / len(df) * 100).round(1)
print(missing_pct[missing_pct > 0])

# Which rows have ANY null?
rows_with_nulls = df[df.isnull().any(axis=1)]
print(f"Rows with at least one null: {len(rows_with_nulls)}")

Step 2 — Decide: Drop or Fill?

SituationWhat to doMethod
< 5% missingUsually safe to drop those rowsdropna(subset=["col"])
Numeric, randomFill with mean (normal) or median (skewed)fillna(df["col"].mean())
CategoricalFill with mode (most frequent) or "Unknown"fillna(df["col"].mode()[0])
> 50% missingDrop the entire column — not usefuldrop(columns=["col"])
Time seriesForward-fill from previous valid valuefillna(method="ffill")

Step 3 — Apply the Fix

python
# Strategy for Titanic dataset:
# Age (19.9% missing) → fill with median
# Cabin (77.1% missing) → drop the column
# Embarked (0.2% missing) → drop those 2 rows

# ── Fill numeric — use median (robust to outliers) ────
age_median = df["Age"].median()
df["Age"] = df["Age"].fillna(age_median)

# ── Drop column — too many nulls (77%) ───────────────
df = df.drop(columns=["Cabin"])

# ── Drop rows — only 2 rows affected ─────────────────
df = df.dropna(subset=["Embarked"])

# Verify — should be 0 for our target columns
print(df[["Age", "Embarked"]].isnull().sum())

# ── Fill categorical with mode ────────────────────────
df["Embarked"] = df["Embarked"].fillna(df["Embarked"].mode()[0])

# ── Fill with custom label ─────────────────────────────
df["Name"] = df["Name"].fillna("Unknown")

print(f"Remaining nulls: {df.isnull().sum().sum()}")

Mean vs Median — Which to use?
Use mean when data is normally distributed (symmetric bell curve). Use median when data has outliers or is skewed — the median is resistant to extreme values. For the Titanic's Age column, a few passengers are very old (max 80), so median is safer. For Fare (extremely skewed — max £512 vs median £14), always use median.
⚠ Warning — the inplace trap
Many tutorials use df.fillna(x, inplace=True). This modifies the original DataFrame directly without returning anything. It's considered bad practice in modern Pandas because it can cause unexpected behaviour. Always prefer the assignment style: df["col"] = df["col"].fillna(x) or df = df.fillna(x).
03 — duplicates

Finding & Removing Duplicates

Duplicate rows corrupt your analysis. Counting customers? Duplicates inflate numbers. Training a model? Duplicates cause overfitting — the model memorises those rows. Always check for and remove them early.

python
# ── Check for duplicates ──────────────────────────────
print(df.duplicated().sum())     # total duplicate rows

# See the actual duplicate rows
print(df[df.duplicated(keep=False)])  # shows ALL copies

# Check duplicates in a specific column (e.g. email)
print(df.duplicated(subset=["PassengerId"]).sum())

# ── Remove duplicates ─────────────────────────────────
df = df.drop_duplicates()     # keep first occurrence

# Keep last occurrence instead
df = df.drop_duplicates(keep="last")

# Remove based on specific columns only
# (e.g., same PassengerId regardless of other cols)
df = df.drop_duplicates(subset=["PassengerId"], keep="first")

# Always reset index after dropping rows!
df = df.reset_index(drop=True)

print(f"Shape after dedup: {df.shape}")
Always reset_index after dropping
When you drop rows, the index becomes non-consecutive (e.g., 0, 1, 5, 8...). This can cause bugs later when you use .iloc or merge DataFrames. Always call df.reset_index(drop=True) after dropping rows. The drop=True prevents the old index from becoming a column.
04 — data types

Fixing Wrong Data Types

Pandas is smart about types but not perfect. When you load a CSV, numbers stored as text come in as object dtype — and you can't do math on them. Dates come in as strings. Categories come in as generic text. Fix them early.

python
# Sample dataframe with type problems
df_messy = pd.DataFrame({
    "price":    ["₹1,200", "₹850",   "₹3,400"],
    "date":     ["2024-01-15", "2024-03-22", "2024-07-01"],
    "gender":   ["Male",     "Female",   "Male"],
    "is_active":["True",     "False",    "True"]
})

print(df_messy.dtypes)  # all "object" — wrong!

# ── Fix 1: String with symbols → numeric ─────────────
df_messy["price"] = (
    df_messy["price"]
    .str.replace("₹", "", regex=False)
    .str.replace(",",  "", regex=False)
    .astype(float)
)

# ── Fix 2: String dates → datetime ───────────────────
df_messy["date"] = pd.to_datetime(df_messy["date"])

# Now you can extract parts!
df_messy["month"] = df_messy["date"].dt.month
df_messy["year"]  = df_messy["date"].dt.year

# ── Fix 3: Low-cardinality text → category ────────────
df_messy["gender"] = df_messy["gender"].astype("category")

# ── Fix 4: "True"/"False" strings → boolean ──────────
df_messy["is_active"] = df_messy["is_active"].map({"True": True, "False": False})

print(df_messy.dtypes)  # now all correct!
print(df_messy["price"].mean())   # 1816.67 — math works!
Output — fixed dtypes
price       float64
date        datetime64[ns]
gender      category
is_active   bool
month       int32

Average price: 1816.67
Why "category" dtype matters
When a text column has only a few unique values (like gender: Male/Female, or Pclass: 1/2/3), converting to category dtype can reduce memory usage by up to 90%. On a dataset with a million rows, this is the difference between 8MB and 800KB for that column. In production ML pipelines with large datasets, this matters enormously.
05 — string cleaning

Cleaning Text & String Columns

Text data is almost always inconsistent. Extra spaces, mixed capitalisation, different abbreviations — all create fake duplicates and ruin groupby operations. Pandas' .str accessor gives you string methods for entire columns at once.

python
df_cities = pd.DataFrame({
    "city": ["  delhi  ", "MUMBAI", "Bangalore",
             "mumbai",   "Delhi",   "BANGALORE"],
    "name": ["  Arjun ", "priya", "ROHAN",
             "Sneha  ",  "dev",   "Ananya"]
})

# Before cleaning — 5 "unique" cities, really only 3
print("Before:", df_cities["city"].nunique())  # 6!

# ── The standard cleaning chain ───────────────────────
df_cities["city"] = (
    df_cities["city"]
    .str.strip()           # remove leading/trailing spaces
    .str.lower()           # lowercase everything
    .str.capitalize()      # Title case: "delhi" → "Delhi"
)

df_cities["name"] = (
    df_cities["name"]
    .str.strip()
    .str.title()           # Title Case each word
)

# After cleaning
print("After :", df_cities["city"].nunique())   # 3! ✓

# More useful .str methods
df["Name"].str.contains("Mrs")      # boolean mask
df["Name"].str.replace("Mr.", "")   # remove prefix
df["Name"].str.split(",")           # split into list
df["Name"].str.len()               # length of each string
df["Name"].str.startswith("A")     # starts with "A"

06 — outliers

Detecting & Handling Outliers

Outliers are values that are far from the rest of the data. They can be genuine (a billionaire in a salary dataset) or errors (age = 999). Detecting them is crucial before training any ML model.

python
# ── Method 1: IQR (Interquartile Range) ───────────────
# The most robust statistical method
Q1 = df["Fare"].quantile(0.25)
Q3 = df["Fare"].quantile(0.75)
IQR = Q3 - Q1

lower = Q1 - 1.5 * IQR
upper = Q3 + 1.5 * IQR

print(f"Fare outlier bounds: £{lower:.2f} to £{upper:.2f}")

outliers = df[(df["Fare"] < lower) | (df["Fare"] > upper)]
print(f"Outlier rows: {len(outliers)}")

# ── Option A: Remove outlier rows ─────────────────────
df_clean = df[(df["Fare"] >= lower) & (df["Fare"] <= upper)]

# ── Option B: Cap (clip) outliers at bounds ───────────
df["Fare_capped"] = df["Fare"].clip(lower=lower, upper=upper)

# ── Method 2: Simple domain-knowledge filter ──────────
# Age can't be negative or over 120
df = df[(df["Age"] >= 0) & (df["Age"] <= 120)]
Remove vs cap — which one?
Remove when the outlier is almost certainly an error (age = 999, negative salary). Cap (clip) when the outlier might be real but you don't want it to distort your model — capping keeps the data point but limits its extreme effect. For the Titanic's Fare column: the £512 fare is real (first class), so capping is better than removal.
07 — complete pipeline

The Complete Cleaning Pipeline

Here is a production-ready cleaning function that combines everything you've learned — the kind of function you'd actually write at the start of a real data science project.

python
import pandas as pd

def clean_titanic(df):
    """Complete Titanic data cleaning pipeline."""
    df = df.copy()    # never modify original!

    print(f"Starting shape: {df.shape}")
    print(f"Initial nulls : {df.isnull().sum().sum()}")

    # ── Step 1: Drop high-null column ─────────────────
    df = df.drop(columns=["Cabin"])

    # ── Step 2: Handle missing values ─────────────────
    df["Age"]      = df["Age"].fillna(df["Age"].median())
    df["Embarked"] = df["Embarked"].fillna(df["Embarked"].mode()[0])
    df["Fare"]     = df["Fare"].fillna(df["Fare"].median())

    # ── Step 3: Remove duplicates ─────────────────────
    n_before = len(df)
    df = df.drop_duplicates()
    print(f"Removed {n_before - len(df)} duplicate rows")

    # ── Step 4: Fix data types ────────────────────────
    df["Sex"]      = df["Sex"].astype("category")
    df["Embarked"] = df["Embarked"].astype("category")
    df["Pclass"]   = df["Pclass"].astype("category")
    df["Survived"] = df["Survived"].astype(bool)

    # ── Step 5: String cleaning ───────────────────────
    df["Sex"] = df["Sex"].str.strip().str.lower()

    # ── Step 6: Cap outliers ──────────────────────────
    Q1, Q3 = df["Fare"].quantile([0.25, 0.75])
    IQR = Q3 - Q1
    df["Fare"] = df["Fare"].clip(
        lower=Q1 - 1.5*IQR, upper=Q3 + 1.5*IQR
    )

    # ── Step 7: Reset index ───────────────────────────
    df = df.reset_index(drop=True)

    print(f"Final shape   : {df.shape}")
    print(f"Remaining nulls: {df.isnull().sum().sum()}")
    return df

# Run it!
url = "https://raw.githubusercontent.com/datasciencedojo/datasets/master/titanic.csv"
raw_df   = pd.read_csv(url)
clean_df = clean_titanic(raw_df)

Always use df.copy()
The first line of any cleaning function should be df = df.copy(). This creates an independent copy so your cleaning function never modifies the original DataFrame. It lets you run the function multiple times, experiment safely, and compare raw vs clean data side by side. This is a professional habit — use it always.
